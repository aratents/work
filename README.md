# מערכת עובדים - ארה אוהלים ומבני מתיחה בעמ

פרויקט Node.js מלא להפעלת פורטל עובדים:
- כניסה לפי תעודת זהות
- שמירת פרטי עובד
- העלאת תעודת זהות ותיאום מס
- קישור לטופס 101
- מילוי שעות עבודה חודשיות
- חישוב שעות רגילות, שעות נוספות, נסיעות וימי עבודה
- שליחת טופס חודשי למייל החברה
- דוח חודשי מרוכז שנשלח למייל החברה אוטומטית

## התקנה

```bash
npm install
cp .env.example .env
npm start
```

המערכת תעלה בכתובת:
`http://localhost:3000`

## פריסה לשרת

מתאים לשרת Node.js רגיל או לשירותים כמו:
- Render
- Railway
- VPS עם Ubuntu + Nginx
- cPanel Node.js App אם קיים באחסון

## משתני סביבה חשובים

מלא בתוך `.env`:
- SMTP_HOST
- SMTP_PORT
- SMTP_SECURE
- SMTP_USER
- SMTP_PASS
- SMTP_FROM
- ADMIN_KEY
- BASE_URL

## איך זה עובד

### צד עובד
העובד נכנס עם תעודת זהות, מעדכן את הפרטים שלו, מעלה מסמכים, ממלא שעות ושולח בסוף חודש.

### צד חברה
כאשר עובד שולח חודש:
1. נשמר snapshot של החודש במסד הנתונים
2. נשלח מייל לחברה עם סיכום
3. נשמר לריכוז חודשי

### איסוף חודשי מרוכז
המערכת מפעילה cron בכל יום בשעה 08:00.
ביום הראשון לכל חודש, אם עדיין לא נשלח ריכוז עבור החודש הקודם, נשלח מייל מרוכז אחד עם CSV של כל השליחות שהגיעו.

## אבטחה
כדי לעמוד בבקשתך, מסך העובד נכנס רק לפי תעודת זהות.
לשימוש אמיתי בפרודקשן מומלץ מאוד להוסיף שכבת אימות נוספת כמו:
- קוד חד פעמי לנייד
- קוד חד פעמי למייל
- סיסמה ראשונית

## נתונים
המערכת שומרת נתונים בתיקיית `data/` בקובץ JSON.
למערכת קטנה זה מספיק.
אם תרצה, אפשר לשדרג בהמשך ל-PostgreSQL / MySQL.

## מסלול אדמין
יש API פנימי בלבד:
`GET /api/admin/monthly-summary?month=YYYY-MM`
עם כותרת:
`x-admin-key: YOUR_ADMIN_KEY`

אפשר להשתמש בו לממשק ניהול עתידי.


## פריסה ל-Railway

### 1. העלאה ל-GitHub
העלה את כל תיקיית הפרויקט ל-repository חדש ב-GitHub.

### 2. יצירת פרויקט ב-Railway
- היכנס ל-Railway
- בחר New Project
- בחר Deploy from GitHub repo
- חבר את המאגר שלך

### 3. הוספת Volume קבוע
מאוד חשוב להוסיף Volume כדי שהנתונים והקבצים לא יימחקו אחרי deploy:
- הוסף Volume לשירות
- הגדר Mount Path:
`/app/data`

### 4. משתני סביבה ב-Railway
הגדר את המשתנים הבאים:
- SMTP_HOST
- SMTP_PORT
- SMTP_SECURE
- SMTP_USER
- SMTP_PASS
- SMTP_FROM
- COMPANY_EMAIL
- ADMIN_KEY
- BASE_URL
- TZ=Asia/Jerusalem

### 5. דומיין
אפשר להשתמש בדומיין הזמני של Railway או לחבר דומיין משלך.
אחרי שיש כתובת סופית, הגדר את `BASE_URL` לכתובת המלאה.

### 6. פקודות
Railway יזהה לבד שזה פרויקט Node.js ויריץ:
- Install: `npm install`
- Start: `npm start`

### הערה חשובה על קבצים ונתונים
המערכת שומרת נתונים ב-`/app/data`.
בלי Volume קבוע של Railway, כל העובדים, הטפסים והקבצים עלולים להימחק בפריסה מחדש.
